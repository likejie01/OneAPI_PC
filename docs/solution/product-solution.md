# OneAPI PC 产品方案

## 1. 产品目标

构建一个 Windows 桌面客户端，面向已登录用户提供统一的 AI 工作台。客户端只保留桌面端最核心的使用路径，围绕三个顶层功能展开：

1. 聊天
2. Codex
3. Claude

同时提供五个业务域：

1. 助手
2. 订阅
3. 钱包
4. 用量
5. 我的

## 2. 产品定位

- 轻量桌面客户端
- 登录后即用
- 远程服务驱动
- 弱 IDE、强集成
- 适合日常 AI 聊天与 CLI 工作流入口

## 3. 核心原则

- 所有页面中文展示
- 所有文件 UTF-8
- Windows 与 Mac 视觉一致
- 不做控制台、多窗口浏览器、多工具面板
- 不改动后端业务语义，只做客户端封装与配置

## 4. 业务结构

### 顶部主功能

- 聊天
- Codex
- Claude

### 左侧边栏

- 助手
- 订阅
- 钱包
- 用量
- 我的

### 业务入口关系

- 聊天页使用助手直接聊天
- 助手是聊天的配置源
- 订阅、钱包、用量、我的共享同一套账户上下文
- Codex、Claude 是轻量工作台，和聊天域并列

## 5. 平台策略

### Windows

- 优先原生安装器交付
- Codex 优先 WSL 路径
- Claude 优先原生命令行路径
- 国内镜像必须可配置

### Mac

- 结构与 Windows 保持接近
- 先交付可安装可运行版本
- 签名与公证可后置，不作为当前阶段阻断项

## 6. 可交付标准

- 可登录
- 可切换 AI
- 可管理助手
- 可购买订阅
- 可查看钱包与用量
- 可新建和查看 key
- 可完成密码二次确认
- 可完成 Codex / Claude 安装配置
- 可完成基础会话工作流
- 可构建、可验证、可打包
